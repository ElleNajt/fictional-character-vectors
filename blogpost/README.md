# Blogpost: Stress-Testing Persona Space with Fictional Characters

The post is in `post.org` (Emacs org-mode with embedded Python). Eval the org file to regenerate all plots.

## Reproducing from scratch

### 1. GPU cluster: generate responses and extract activations

These Slurm jobs run on a cluster with GPUs. They generate responses to questions and extract internal activations from Qwen3-32B.

```bash
# On the cluster (adjust paths in scripts as needed):
cd /workspace-vast/lnajt

# Character responses + activations (240-question psychographic battery)
sbatch scripts/slurm_monitor_axis.sh

# OOD prompt activations (16 degenerate/task questions)
sbatch scripts/slurm_degenerate_questions.sh

# BiGGen-Bench character activations (90 diverse tasks)
sbatch scripts/slurm_biggen_bench.sh

# BiGGen-Bench role activations (275 Lu et al. roles × 90 tasks)
sbatch scripts/slurm_roles_biggen.sh
```

After jobs complete, pull outputs locally:
```bash
scp -r cluster:outputs/qwen3-32b_20260211_002840/{vectors,activations,responses} outputs/qwen3-32b_20260211_002840/
scp -r cluster:outputs/degenerate_questions/activations outputs/degenerate_questions/
scp -r cluster:outputs/biggen_bench/activations outputs/biggen_bench/
scp -r cluster:outputs/roles_biggen/activations outputs/roles_biggen/
```

### 2. Local: assemble all data files

```bash
git submodule update --init
bash blogpost/scripts/assemble_all.sh          # pure computation steps
bash blogpost/scripts/assemble_all.sh           # full pipeline (needs ANTHROPIC_API_KEY_BATCH)
```

Use `--no-llm` to skip steps that require the Anthropic API (feature discovery and coding). The LLM-generated result files are checked into git, so you can skip these and still eval the post.

### 3. Eval the post

Open `post.org` in Emacs and run all src blocks, or:
```bash
emacsclient --eval '(ob-python-extras-execute-org-file "'$(pwd)'/blogpost/post.org")'
```
