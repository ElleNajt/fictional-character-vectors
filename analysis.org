* Overview

We extracted activations from Qwen 2.5 32B for 399 fictional characters. Each character was prompted with 240 questions, and we averaged the layer 32 activations across all questions to get a single 5120-dimensional vector per character.

PCA on this space reveals interpretable axes of variation.

This work builds on the methodology from [[https://arxiv.org/abs/2601.10387][Lu et al. (2026), "The Assistant Axis: Situating and Stabilizing the Default Persona of Language Models"]], which extracted persona vectors for 275 generic character archetypes.

* Dataset: Canonical Fictional Characters

The character dataset was curated to cover culturally significant fictional characters across diverse sources. The selection affects results - certain clusters emerge because of deliberate inclusion choices.

** How Dataset Choices Affect Results

The dataset was *intentionally* curated to include diversity, which directly affects the PCA results:

- *Indian mythology cluster*: We specifically included 17 characters from the Mahabharata and Ramayana. This creates a visible "Indian Epic" cluster in residual PC1/PC2. Without this deliberate inclusion, this dimension would not appear.

- *Anime representation*: The 28 Japanese anime/manga characters create a distinct cluster. A Western-only dataset would miss this axis.

- *African folklore*: Including Anansi, Sundiata, Mami Wata, and Okonkwo ensures non-Western narrative traditions are represented, though they're a smaller cluster.

- *Greek mythology*: The 18 Greek mythological figures anchor the "divine/cosmic" end of several PCs.

If you replicate this with a different character set, expect different clusters. The PCA finds structure in *whatever characters you give it* - the interpretations depend on the dataset composition.

from scipy.spatial.distance import cdist

def find_similar(query_name, top_k=10):
    """Find characters most similar to query in PCA space."""
    # Find query index
    query_idx = None
    for i, name in enumerate(char_names):
        if query_name.lower() in name.lower():
            query_idx = i
            break
    
    if query_idx is None:
        print(f"Character '{query_name}' not found")
        return
    
    query_vec = transformed[query_idx:query_idx+1]
    distances = cdist(query_vec, transformed, metric='euclidean')[0]
    
    sorted_idx = np.argsort(distances)
    
    print(f"\nCharacters most similar to {char_names[query_idx].replace('_', ' ').title()}:\n")
    for i in sorted_idx[1:top_k+1]:  # Skip self
        name = char_names[i].replace('_', ' ').title()
        print(f"  {name:35s} (distance: {distances[i]:.1f})")

** Character Arithmetic

Like word2vec's "king - man + woman = queen", we can do character arithmetic.

#+begin_src python
def char_arithmetic(positive, negative, top_k=5):
    """
    Find characters closest to: sum(positive) - sum(negative)
    
    Example: villain + comedy = ?
    """
    result_vec = np.zeros(transformed.shape[1])
    
    used_indices = set()
    
    for name in positive:
        for i, cname in enumerate(char_names):
            if name.lower() in cname.lower():
                result_vec += transformed[i]
                used_indices.add(i)
                break
    
    for name in negative:
        for i, cname in enumerate(char_names):
            if name.lower() in cname.lower():
                result_vec -= transformed[i]
                used_indices.add(i)
                break
    
    distances = cdist(result_vec.reshape(1, -1), transformed, metric='euclidean')[0]
    sorted_idx = np.argsort(distances)
    
    pos_str = ' + '.join(positive)
    neg_str = ' - '.join(negative) if negative else ''
    equation = pos_str + (' - ' + neg_str if neg_str else '')
    
    print(f"\n{equation} = ?\n")
    count = 0
    for i in sorted_idx:
        if i not in used_indices:
            name = char_names[i].replace('_', ' ').title()
            print(f"  {name:35s} (distance: {distances[i]:.1f})")
            count += 1
            if count >= top_k:
                break

# Darth Vader - evil = ? (should give heroic Star Wars characters)
char_arithmetic(["darth_vader"], ["voldemort"])

    """Get centroid of franchise characters in PCA space."""
    indices = []
    for fc in franchise_chars:
        for i, name in enumerate(char_names):
            if fc.lower() in name.lower():

import torch
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
from huggingface_hub import hf_hub_download, list_repo_files

# Check for pre-computed role PCA, or download and compute
role_pca_path = Path('data/role_vectors/qwen-3-32b_pca_layer32.pkl')

if role_pca_path.exists():
    # Load pre-computed
    with open(role_pca_path, 'rb') as f:
        role_data = pickle.load(f)
    role_pca = role_data['pca']
    role_scaler = role_data['scaler']
    roles_flat = role_data['role_names']
    role_projections = role_data['transformed']
    print(f"Loaded pre-computed role PCA from {role_pca_path}")
else:
    # Download from HuggingFace and compute PCA
    print("Downloading role vectors from HuggingFace...")
    all_files = list_repo_files("lu-christina/assistant-axis-vectors", repo_type="dataset")
    role_files = [f for f in all_files if f.startswith("qwen-3-32b/role_vectors/")]
    
    role_vectors = {}
    for file_path in role_files:
        role_name = Path(file_path).stem
        local_path = hf_hub_download(
            repo_id="lu-christina/assistant-axis-vectors",
            filename=file_path,
            repo_type="dataset"
        )
        data = torch.load(local_path, map_location="cpu", weights_only=False)
        if isinstance(data, dict) and 'vector' in data:
            role_vectors[role_name] = data['vector']
        elif isinstance(data, torch.Tensor):
            role_vectors[role_name] = data
    
    print(f"Downloaded {len(role_vectors)} role vectors")
    
    # Extract layer 32 and fit PCA
    roles_flat = sorted(role_vectors.keys())
    layer_vectors = []
    for role in roles_flat:
        vec = role_vectors[role]
        if vec.dim() == 2:
            layer_vectors.append(vec[32].float().numpy())
        else:
            layer_vectors.append(vec.float().numpy())
    
    X = np.vstack(layer_vectors)
    role_scaler = StandardScaler()
    X_scaled = role_scaler.fit_transform(X)
    
    role_pca = PCA(n_components=min(X.shape))
    role_projections = role_pca.fit_transform(X_scaled)
    
    print(f"Fitted PCA: {role_pca.n_components_} components")

print(f"Loaded {len(roles_flat)} roles")
print(f"Role PCA components: {role_pca.components_.shape}")

# Project characters onto role space
chars_scaled = role_scaler.transform(activation_matrix)
chars_in_role_space = role_pca.transform(chars_scaled)

# Reconstruct and get residuals
reconstructed = chars_in_role_space @ role_pca.components_
residuals = chars_scaled - reconstructed

# Variance accounting
original_var = np.var(chars_scaled, axis=0).sum()
residual_var = np.var(residuals, axis=0).sum()
captured_var = original_var - residual_var

print(f"\nVariance decomposition:")
print(f"  Total character variance: {original_var:.1f}")
print(f"  Captured by role space:   {captured_var:.1f} ({captured_var/original_var:.1%})")
print(f"  Residual (not captured):  {residual_var:.1f} ({residual_var/original_var:.1%})")

- The role space successfully captures moral and archetypal dimensions

#+begin_src python
def show_residual_pc(pc_idx):
    """Show what's in the residual PCs - structure the role space misses."""
    scores = residuals_transformed[:, pc_idx]
    sorted_idx = np.argsort(scores)
    var_pct = residual_pca.explained_variance_ratio_[pc_idx] * 100
    
    print(f"\n{'='*70}")
    print(f"Residual PC{pc_idx+1} ({var_pct:.1f}% of residual variance)")
    print(f"{'='*70}")
    print("\nHigh:")
    for i in sorted_idx[-7:][::-1]:
        print(f"  {char_names[i].replace('_', ' ').title():35s} {scores[i]:+.1f}")
    print("\nLow:")
    for i in sorted_idx[:7]:
        print(f"  {char_names[i].replace('_', ' ').title():35s} {scores[i]:+.1f}")

for i in range(5):
    show_residual_pc(i)

* Limitations and Caveats

** Franchise Clustering: Validated by Control

We ran a one-per-franchise control (see above). The same PCs emerge with nearly identical variance, suggesting the structure is real, not an artifact of including multiple characters per franchise.

** The Specific PCs Depend on Dataset Composition

The particular axes we found (PC1 = groundedness, PC7 = fairy tale vs Indian epic) are contingent on our character selection. Different results to expect with:

- *More anime characters*: Might see anime-specific PCs (shonen vs slice-of-life, etc.)
- *More African folklore*: Currently underrepresented; would likely form its own cluster
- *Historical figures instead of fictional*: Different archetypal structure entirely

** Single Model, Single Layer

We only analyzed Qwen 2.5 32B at layer 32. Open questions:

- Do other layers show different structure?
- Do other models (Llama, GPT) produce similar PCs?
- Is layer 32 special or would any middle layer work?

** No Ground Truth

We interpret PCs post-hoc based on which characters load high/low. These interpretations are subjective and could be confabulated. The "groundedness" interpretation for PC1 is a hypothesis, not a verified fact.

* Future Experiments

Potential follow-ups that would strengthen or challenge these findings:

1. *Dense franchise sampling*: Collect 50+ characters from a single franchise (e.g., Naruto, Harry Potter, Marvel) and run within-franchise PCA. What structure emerges? Hero/villain? Protagonist/supporting? Generations? This would test whether franchise-specific directions encode meaningful within-universe structure, not just "this is from Naruto."

2. *Layer sweep*: Extract activations at layers 16, 24, 32, 40 and compare PCA structure

3. *Cross-model comparison*: Run same characters through Llama 3, GPT-4, compare PC structure

4. *Activation patching*: Can we make Harry Potter "more Star Wars" by adding the franchise direction?

5. *Random Wikidata sample*: Use top-N by Wikipedia language count without curation
