#+TITLE: Fictional Character Vectors
#+AUTHOR: Elle

PCA analysis of fictional character activations in Qwen 2.5 32B, extending the methodology from [[https://arxiv.org/abs/2601.10387][Lu et al. (2026) "The Assistant Axis"]].

* Quick Start (Pre-computed Results)

The easiest way to explore the results:

#+begin_src bash
git clone https://github.com/ElleNajt/fictional-character-vectors
cd fictional-character-vectors

# Setup
uv venv && uv pip install -e .

# Open analysis notebook
emacs analysis.org
#+end_src

The ~results/fictional_character_analysis.pkl~ contains pre-computed PCA results for 399 characters. Run the code blocks in ~analysis.org~ to explore.

* Key Findings

- *PC1 (18%)*: Groundedness - human/relatable characters vs divine/mythological beings
- *PC2 (8%)*: Moral alignment - villains vs innocents
- *Franchise clustering*: Star Wars, Harry Potter, anime characters cluster together
- *Orthogonal to roles*: Generic role vectors (from Assistant Axis) only explain 43% of character variance

** What Role Space Captures vs Misses

Role vectors capture *archetypal* dimensions:
- Eldritch/cosmic vs analytical/rational
- Slacker/comic vs spiritual/divine
- Malevolent vs nurturing

But they miss *cultural/franchise identity*:
- Star Wars characters cluster beyond "villain" or "hero"
- Indian epic heroes share structure beyond "warrior"
- Anime has a distinct signature

*Characters are more than the sum of their roles*.

* Full Reproduction (Requires GPU)

To reproduce character activations from scratch:

** Requirements
- GPU with ~80GB VRAM (e.g., A100 80GB) for Qwen 2.5 32B
- Python 3.10+
- [[https://github.com/astral-sh/uv][uv]] package manager

** Step 1: Collect Characters

#+begin_src bash
python src/data_collection/fetch_wikidata_characters.py \
    --output data/wikidata_characters.json \
    --min_sitelinks 10

python src/data_collection/merge_character_lists.py \
    --wikidata data/wikidata_characters.json \
    --curated data/curated_characters.json \
    --output data/canonical_fictional_characters.json
#+end_src

** Step 2: Generate Role Files

#+begin_src bash
python src/data_collection/convert_characters_to_roles.py \
    --characters data/canonical_fictional_characters.json \
    --output_dir outputs/character_roles
#+end_src

** Step 3: Generate Responses (GPU)

Uses the [[https://github.com/safety-research/assistant-axis][assistant-axis]] pipeline. The extraction questions (240 diverse prompts) come from that repo.

#+begin_src bash
# Clone assistant-axis for the full pipeline
git clone https://github.com/safety-research/assistant-axis
cd assistant-axis && uv sync && cd ..

# The questions file is in assistant-axis/data/extraction_questions.jsonl
uv run python src/extraction/1_generate.py \
    --model Qwen/Qwen2.5-32B-Instruct \
    --roles_dir outputs/character_roles \
    --questions_file assistant-axis/data/extraction_questions.jsonl \
    --output_dir outputs/responses \
    --num_workers 4
#+end_src

** Step 4: Extract Activations (GPU)

#+begin_src bash
uv run python src/extraction/2_activations.py \
    --model Qwen/Qwen2.5-32B-Instruct \
    --responses_dir outputs/responses \
    --output_dir outputs/activations \
    --num_workers 4
#+end_src

** Step 5: Run PCA

#+begin_src bash
uv run python src/analysis/fictional_character_pca.py \
    --activation-dir outputs/activations \
    --output-dir results \
    --layer 32 \
    --n-components 50
#+end_src
