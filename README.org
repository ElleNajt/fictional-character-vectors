PCA analysis of fictional character activations in Qwen 3 32B, extending the methodology from [[https://arxiv.org/abs/2601.10387][Lu et al. (2026) "The Assistant Axis"]].

* Reproducing from Scratch

This repo is designed to be fully reproducible. Clone it, give it a GPU machine, and regenerate the data.

** Requirements

- GPU cluster with 4x 80GB GPUs (e.g., 4x A100/H100) for Qwen 3 32B with tensor parallelism
- Python 3.10+
- SLURM (for cluster) or local GPU setup

** Step 1: Clone with Submodule

#+begin_src bash
git clone --recurse-submodules https://github.com/ElleNajt/fictional-character-vectors
cd fictional-character-vectors
#+end_src

** Step 2: Prepare Role Files

Convert character JSONs to role instruction format:

#+begin_src bash
python scripts/prepare_roles.py
#+end_src

This creates ~data/roles/instructions/*.json~ (914 role files from 6 franchises + canonical characters).

** Step 3a: Run on SLURM Cluster

#+begin_src bash
# Submit the extraction job
sbatch scripts/slurm_extract.sh
#+end_src

The job will:
1. Create a virtual environment (first run only)
2. Install dependencies (first run only)
3. Generate 240 responses per character using Qwen 3 32B
4. Extract layer 32 activations from all responses
5. Save to ~outputs/qwen3-32b_<timestamp>/~

Monitor progress:
#+begin_src bash
# Check job status
squeue -u $USER

# Tail the log
tail -f logs/char_vectors_<jobid>.out
#+end_src

** Step 3b: Run Locally (Interactive)

For a local GPU machine:

#+begin_src bash
./scripts/run_extraction.sh --model Qwen/Qwen3-32B --gpus 4
#+end_src

** Step 4: Run PCA Analysis

Once extraction is complete:

#+begin_src bash
python src/analysis/fictional_character_pca.py \
    --activation-dir outputs/qwen3-32b_<timestamp>/activations \
    --output-dir results \
    --layer 32
#+end_src

* Dataset

914 fictional characters from:
- 6 dense franchise samples (50-100+ characters each):
  - Harry Potter (102)
  - Naruto (84)
  - Marvel (102)
  - Star Wars (103)
  - Game of Thrones (99)
  - Lord of the Rings (97)
- Canonical fictional characters (327) from diverse global sources

Role prompts are minimal: just character name and source (e.g., "You are Harry Potter from Harry Potter."). We intentionally omit descriptions to let the model's own representation emerge.

* Key Findings

- *PC1 (18%)*: Groundedness - human/relatable characters vs divine/mythological beings
- *PC2 (8%)*: Moral alignment - villains vs innocents
- *Franchise clustering*: Characters from the same franchise cluster together
- *Orthogonal to roles*: Generic role vectors only explain 43% of character variance

*Characters are more than the sum of their roles.*

#+begin_src bash
#SBATCH --partition=your-partition

#SBATCH --exclude=bad-nodes         # Exclude problematic nodes
#+end_src
